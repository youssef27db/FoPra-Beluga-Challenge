25class BelugaGymCompatibleDomain(D):
26 """This abstract scikit-decide domain allows the scikit-decide library to automatically
27 transforms one of the Beluga scikit-decide domains into a gymnasium environment via the
28 BelugaGymEnv class below. Indeed, the 3 Beluga scikit-decide domains, namely SkdPDDLDomain,
29 SkdPPDDLDomain and SkdSPDDLDomain, use PDDL-style states and actions that cannot be readily
30 understood by reinforcement learning algorithms. We must transform those states and actions
31 in a vectorized form (e.g. numpy arrays), which is the intent of this BelugaGymCompatibleDomain
32 class.
33
34 Please note that there is no bias in the decision to model the problem using PDDL domains at first.
35 The reason for this choice is that the industrial decision-making problem at stake is naturally
36 structured using predicates and fluents as in PDDL, and so is the transition logics of the system.
37
38 However, there are many good and poor ways to transform PDDL-style states and actions into tensors.
39 This is why this domain class is abstract: it has to be specialized to implement the methods responsible
40 for translating from PDDL semantics to tensors and vice versa. The generate_solve_rllib_test.py script
41 provides a simplistic example of such a specialized domain class.
42 """
